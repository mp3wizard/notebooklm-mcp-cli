# v0.8.3 Maintenance Release

This maintenance release adds an experimental opt-in browser-backed transport
for NotebookLM auth replay failures reported in issue #248.

## Added

- Added `NOTEBOOKLM_RPC_TRANSPORT=cdp` to route normal batchexecute RPCs through
  an in-page `fetch` inside the saved NotebookLM browser profile.
- Added CDP transport support for notebook chat via `GenerateFreeFormStreamed`,
  including longer CDP response waits for long answers.
- Added focused tests for CDP transport routing, streamed query routing, long
  CDP timeouts, and cleanup of browser instances launched by the client.
- Documented when and how to use the experimental transport.

## Changed

- Reused the new CDP fetch helper inside `nlm doctor auth-replay`, so the
  diagnostic path matches the real experimental transport more closely.
- Kept the default `httpx` transport unchanged. The CDP path is off by default.

## Notes

Use `nlm doctor auth-replay` first. Only try
`NOTEBOOKLM_RPC_TRANSPORT=cdp` when normal replay fails but `cdp_in_page`
passes. Uploads, downloads, and artifact file transfers still use the existing
HTTP paths.
